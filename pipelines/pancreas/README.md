# Pancreas Cohort Pipeline — Forthcoming

**Status:** stub. The pancreas-cohort pipeline (planned third disease in
the diabetes / lung-cancer / pancreas trio that backs the multi-agent
proposal) is not yet implemented. This folder is a placeholder so the
top-level layout reflects the intended scope.

## Plan

Adapter from the [lung-cancer template](../lung_cancer/), with these
disease-specific changes:

| Layer | Change from `pipelines/lung_cancer/` |
|---|---|
| Cohort anchor | PhecodeX `CA_101.5` (malignant neoplasm of the pancreas) — replaces `CA_102.1` (lung). |
| ICD reference | `pancreas_cancer_codes` table, populated by an updated `build_reference_tables.sql` that pulls `C25.*` (ICD-10) + `157.*` (ICD-9) children from `phecodeX_unrolled_ICD_CM` plus the optional clinician-exclusion siblings (e.g. neuroendocrine `CA_101.51`). |
| Treatment HCPCS | Pancreatic-cancer-specific chemotherapy J-codes (gemcitabine — J9201; FOLFIRINOX components — irinotecan J9206, oxaliplatin J9263, fluorouracil J9190; nab-paclitaxel J9264). Smaller immunotherapy footprint than lung — checkpoint inhibitors are approved for MSI-H/dMMR pancreatic tumors but not as standard first-line therapy. |
| Outcomes | Same 5-stage shape (cohort → exposure → outcome → demographics+criteria → final). The autoimmune-events outcome layer ports unchanged. |
| Validation | Cohort size will be much smaller than lung (pancreatic cancer incidence is ~1/8 of lung). Synthea's pancreatic-cancer modeling is even sparser than lung, so the synthetic-data oncology overlay in [`../../synthetic_data/load_rif.py`](../../synthetic_data/load_rif.py) will need a parallel pancreas branch. |

## When to populate this folder

Two reasonable triggers:

1. A specific protocol question that needs the pancreas pipeline (then
   build it scoped to that question, not a general pipeline).
2. A reviewer or workshop poster that needs the third disease as proof
   of the cross-disease generalisation claim — at which point an
   adapter from `pipelines/lung_cancer/` is the cheapest path
   (~80% of the SQL is identical; the disease anchors and treatment
   HCPCS lists change).

The agent project's premise is precisely that this adapter should be
generated by the multi-agent prototype itself when given a natural-
language pancreas-cancer protocol description, not hand-coded each
time. Populating this folder by hand is therefore a contingency, not
the primary plan.
