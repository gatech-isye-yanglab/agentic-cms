## The method (from the a legacy commercial-claims study study)

The autoimmune ICD table is built by the **same PheWAS-anchor recipe** as
the lung-cancer table, but using *multiple* PheWAS codes corresponding to
the Super-PheWAS autoimmune category tree.

General recipe (same three moves as `lung_cancer_icd_legacy_claims.md`):

1. **Anchor:** a set of PheWAS codes covering the autoimmune spectrum —
   endocrine autoimmune (type-1 diabetes, thyroiditis, hypophysitis),
   rheumatologic (rheumatoid arthritis, lupus, psoriasis), dermatologic
   (vitiligo, bullous pemphigoid), gastrointestinal (colitis, hepatitis),
   neurologic (myasthenia gravis, Guillain-Barré), pulmonary
   (pneumonitis), renal (nephritis), hematologic (hemolytic anemia,
   thrombocytopenia). the published thesis Appendix C calls this the
   "Super-PheWAS autoimmune category" set — roughly 56 categories.
2. **Pull:** ICD-9 and ICD-10 children of each PheWAS anchor from
   `PheWAS.dbo.Icd9CodeTranslation` and `PheWAS.dbo.Icd10CodeTranslation`.
3. **Exclude:** "Personal history" rows (V-codes and Z-codes). In the
   autoimmune context there is an additional exclusion: **`V12.x`
   personal-history-of-disease codes** must also be dropped because they
   mark prior autoimmune diagnoses rather than current events.

A further filter tracks which category each ICD belongs to, because the
downstream analysis needs per-category Cox models (see
`../step5_consolidate/final_tables.sql` procedures `immuno_table_dm`,
`immuno_table_hypo`, `immuno_table_ra` — each filters to one Super-PheWAS
category).

## Disease-subgroup sub-slices

The `step5_consolidate/final_tables.sql` procedures use three named
sub-slices of `autoimmune_icd`:

| Sub-slice | Super-PheWAS category | Used by |
|---|---|---|
| `autoimmune_icd_dm` | Type-1 diabetes (PheWAS 250.1x family) | `immuno_table_dm` |
| `autoimmune_icd_hypo` | Hypophysitis / endocrine pituitary autoimmune | `immuno_table_hypo` |
| `autoimmune_icd_ra` | Rheumatoid arthritis (PheWAS 714.x) | `immuno_table_ra` |
| (`autoimmune_icd_thyroiditis`, `_myalgia` — thesis only) | Thyroiditis, polymyalgia rheumatica | thesis analysis |

Agent pattern: each sub-slice is the full `autoimmune_icd` table filtered
to one Super-PheWAS category. The agent should produce `autoimmune_icd`
once with a category column, and derive the sub-slices as one-line `WHERE
super_phewas_category = 'X'` filters — not as six separately hand-curated
tables.

## Clinical refinements from the a legacy commercial-claims study study

Ken Kehl's Nov 22, 2017 feedback (from a legacy commercial-claims study-paper research log
`a legacy commercial-claims studyLungCancer/NOV2017/nov22.tex`, not in this repo; content transcribed
below):

- **Look-back window.** "Excluding patients with pre-existing autoimmune
  diseases: eight year window is plenty, usually looking back a year or
  two would be enough." → The inclusion rule "no autoimmune diagnosis in
  the year before lung-cancer diagnosis" is the a legacy commercial-claims study study's operational
  clean-exposure rule.
- **Autoimmune adverse event ≠ autoimmune disease.** "Autoimmune adverse
  event is different from autoimmune disease, overlap but different. Need
  to develop a way to identify immune-related adverse event, maybe using
  combinations of diagnosis code and treatment." → the CMS pipeline
  resolved this by requiring the autoimmune event to occur *after* the
  immunotherapy start date, and by combining diagnosis with hospitalization
  and treatment signals.
- **Adverse event definition.** "Hospitalization with steroid" as a
  confirmatory combined signal — also used bythe gold-standard authorin the inpatient flavor
  of `autoimmune_inpatient_records[_v2]`.

## Agent-benchmark framing

Given NL prompt *"find immune-related adverse events after immunotherapy"*,
the agent must:

1. Enumerate the Super-PheWAS autoimmune categories (clinical taxonomy
   step — this is where a clinician either approves the standard ~56-
   category set or narrows it).
2. For each PheWAS anchor code, pull ICD-9 and ICD-10 children.
3. Exclude "Personal history" and `V12.x` / `Z86.x` codes.
4. Label each ICD with its Super-PheWAS category so that downstream
   sub-slices (`_dm`, `_hypo`, `_ra`, …) can be derived by one-line
   filters.

the published thesis tables are the benchmark. The agent's output is scored as
per-category precision/recall against those tables. The ~56-category rollup
is the critical learning target — a simpler agent that does one-code-in,
one-code-out lookup will not reproduce it.
